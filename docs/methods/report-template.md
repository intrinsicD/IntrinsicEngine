# Method Report Template

Use this structure for per-method validation and performance reports.

## 1. Method identity

- Method ID
- Paper title/authors/year
- Commit and environment

## 2. Backend matrix

- CPU reference status
- CPU optimized status
- GPU backend status (if present)

## 3. Correctness summary

- Analytic/simple-case test results
- Regression test results
- Parity metrics vs reference backend

## 4. Benchmark summary

- Benchmark IDs executed
- Runtime/memory/quality metrics
- Baseline comparison notes

## 5. Numerical diagnostics

- Degenerate-case behavior
- Failure modes and mitigation
- Known limitations

## 6. Follow-up actions

- Open issues/tasks
- Validation gaps
